Really Simple History Testing Instructions

This utility is designed to test the functionality of Really Simple History's historyStorage and dhtmlHistory objects.

Use the links above to:

  • Add history events one at a time and see them logged in the dhtmlHistory log window.
  • Navigate off-site so you can then return with the back button and confirm your history was retained.
  • Run a large suite of historyStorage tests and view the results in the historyStorage log window.
  • Run a separate historyStorage test to see how it handles serialization of complex XML data loaded via XHR.
  • Send dhtmlHistory and historyStorage into debug mode, where all hidden DOM elements get unhidden. This allows you to track the library's behind-the-scenes activities by peering into the textarea that serves as its storage bin and the Safari- and IE-only form fields that help overcome those browsers' peculiarities.
  • Print your browser stats to yet another log window so you can report browser-specific bugs over in the Google Code issue tracker.

CAVEAT:Don't try to run this test page, or RSH in general, from your desktop. Always view it from a webserver (even localhost); otherwise its behavior will be extremely flaky, especially in IE.
